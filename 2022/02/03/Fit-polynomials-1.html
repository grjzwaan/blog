Introduction

Fitting with polynomials is the natural step-up from linear regression: fitting with a line. This article is on interpolating series and functions with polynomials, both exact and approximately.

What started as a small idea to generate a moving image of a polynomial chasing random points has evolved in a tour through nice mathematics. There is both code and mathematics so pick whatever suits you. I hope you enjoy it!

We start with defining the problem and some notation. No worries, we also add graphs to make the problem explicit.

Example inputs

We can either be given some points $p_1,...,p_n$ or be given a function $f()$ that we want to approximate. I use these interchangably, because a function is convenient as ground truth and to approximate a function we will be sampling from that function.

To make it concrete lets use some functions and sample 5 points $x_1,...,x_5$ from $[-1, +1]$. Additionally, we use the diabetes dataset from SciKit.

Interpolation vs fit

The goal is to recover the ground truth, given the samples. As can be seen in the examples this is possible for the first few examples, but for the diabetes example there are multiple values for the same $x$. Let's make it concrete what we're trying to do.

We are given points $x_1,...,x_n$ and values $y_1,...,y_n$ and we want to select a polynomial function $p(x)$ of degree $k$:

$$ p_k(x) = c_0 x^0 + c_1 x^1 + ... + c_k x^k $$

For interpolation we want that $p(x_i)=y_i$ for $i=1,...n$.

For approximation we want to find a polynomial that approximates the value. For this we take the $L_2$ or squared difference distance:

$$ \min_{p_k(x)} \sum_i ( y_i - p_k(x_i) )^2 = \min_{p_k(x)} || y_i - p_k(x_i) ||_2 $$

Matrix notation

It is quite convenient to write the equations above in matrix form and make the coefficients $c$ explicit. In the optimization we keep degree $k$ fixed, but have to pick the best $c$.

We can do this in matrix form and state $M_k(x)c=y$, where $M_k(x)$ is the matrix for the values $x$ and the powers up to $k$ (the monomials):

For $k=2$ it would be \begin{equation*} M_2(x) = \begin{pmatrix} 1 & -1 & 1 \\ 1 & -.5 & .25 \\ 1 & 0 & 0 \\ 1 & .5 & .25 \\ 1 & 1 & 1 \\ \end{pmatrix} \end{equation*}

For approximation we would write

$$ min_c ||y - M_k(x)c||_2 $$

The matrix of the monomials is called the Vandermonde matrix. Using the functionality in NumPy gives the same matrix as above:

Interpolation of a polynomial with monomials

For $n$ points there exists a unique polynomial of degree $n-1$ that passes through these points. To find this polynomial we can solve a linear system of $n$ equations with $n$ variables.

Example for n=2

As an example take the points $(-0.5,0.25), (0, 0), (0.5, 0.25)$ and we want to find a quadratic polynomial $p(x) = c_0 + c_1 \cdot x + c_2 \cdot x^2$.

The linear equations will be $$ \begin{align} p(-0.5) &= 0.25 = c_0 + c_1 \cdot -0.5 + c_2 \cdot (-0.5)^2 \\ p(0) &= 0 = c_0 + c_1 \cdot 0. + c_2 \cdot 0^2 \\ p(0.5) &= 0.25 = c_0 + c_1 \cdot 0.5 + c_2 \cdot (0.5)^2 \end{align} $$

For this example we can solve by hand by noting that $c_0=0$ (second equation), which leaves $$ 0.25 = -c_1/2 + c_2/4 $$ $$ 0.25 = c_1/2 + c_2/4 $$

and thus $c_1=0$ and $c_2=1$.

This is a good thing since I picked the points from the function $f(x) = x^2$ :).

General

In general we can do this with the matrix of monomials (Vandermonde matrix) and solving the linear equation.

We can do this in matrix form and state $M_{n-1}(x)c=y$, where $M_{n-1}(x)$ is the Vandermonde matrix for the values $x$ and the powers.

For the previous example it would be \begin{equation*} A_2(x) = \begin{pmatrix} 1 & -\frac{1}{2} & \frac{1}{4} \\ 1 & 0 & 0 \\ 1 & \frac{1}{2} & \frac{1}{4} \\ \end{pmatrix} \end{equation*}

Then we solve $Ac=y$ for $c$. For example by taking the inverse $A^{-1}y=c$.

A = np.array([[1, -1/2, 1/4], [1, 0, 0], [1, 1/2, 1/4]])
y = np.array([0.25, 0., 0.25])
c = np.linalg.inv(A) @ y
c

array([0., 0., 1.])

We can do the same programmatically for the Runge function and degree $4$

def interpolate(f, x):
    M = np.vander(x, increasing=True)
    c = np.linalg.inv(M) @ f(x)
    return c

Aside: solve instead of invert

Generally it is better to solve for $c$ instead of taking the explicit inverse. For example, by finding the LU-decomposition $M=LU$ and then solve $Lz=c$ and then $Uc = z$. Since $L$ and $U$ are triangular matrices this is straightforward.

The NumPy function solve does this with the method from LAPACK:

x = np.array([-.5, 0., .5])
M = np.vander(x, increasing=True)
c_2 = np.linalg.solve(A, f(x))
c_2

array([0., 0., 1.])

The difference between the two method is due to the number of floating point operations and the amount of error introduced. We will show this with the Hilbert matrix as an example. The Hilbert matrix is ill conditioned, small changes in the matrix will lead to large changes in the computed output. (Note: The matrix $H$ above is almost the Hilbert matrix.)

Define the Hilbert matrix

def Hilbert(n):
    return np.array([ [(1/(i+j-1)) for j in range (1, n+1)] for i in range (1, n+1)])

The inverse of the Hilbert matrix is known in closed form (see https://en.wikipedia.org/wiki/Hilbert_matrix):

Define the error as the mean squared difference between solving through taking the inverse and solving throuh LU-decomposition:

def error(n):
    x = np.linspace(0, 1, n)
    f = lambda x: x**2
    y = f(x)
    H = Hilbert(n)

    c_inverse = np.linalg.inv(H)@y
    c_solve = np.linalg.solve(H, y)
    error = np.mean((c_inverse - c_solve)**2)
    return error

You can see that the difference is quite pronounced for large $n$

Approximation

What if we wanted to approximate with a lower degree polynomial than the number of points? For this we can use the Moore-Penrose inverse, a generalization of the normal matrix inverse $A A^{-1} = I$

The Moore-Penrose inverse (Denoted as $A^+$) can be defined in terms of solving a linear system of equations $Ac=y$. The solution to $A^+y = \hat{c}$ is the least squares solution i.e. $||Ac-y||_2 \geq ||A\hat{c}-y||_2$.

With this result we can approximate

def approximate(y, x, d=3):
    A = np.vander(x, N=d, increasing=True)
    c_approx = np.linalg.pinv(A) @y
    return c_approx

A good question, that I do not know the answer too, is how the numerical instability (condition number) of the Moore-Penrose inversion relations to the regular inversion. If you know: please tell me.
